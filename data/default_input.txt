# ODOO ERP CUSTOM DEVELOPMENT PROJECT
# โครงการพัฒนา ODOO ERP แบบปรับแต่ง

## PROJECT OVERVIEW | ภาพรวมโครงการ
ให้นักศึกษาฝึกงาน 7 คน ร่วมกันพัฒนาโปรแกรม ERP ODOO community version 17 โดยการแบ่งหน้าที่ตามที่ได้รับการจัดสรรตามความสนใจและความถนัด

Development team consists of 7 intern students working together to develop custom addons for ODOO community version 17, with responsibilities assigned according to their interests and skills.

## PRIMARY REQUIREMENTS | ความต้องการหลัก
พัฒนาโดยการเพิ่ม custom addons เพื่อแก้ไขหรือเพิ่ม feature ให้กับ ODOO โดยมี features หลักที่เป็น requirement อยู่ 2 อย่าง ดังนี้:

The project involves developing custom addons to modify or add features to ODOO with two primary requirements:

### 1. OAuth Integration | การเชื่อมต่อ OAuth
การเพิ่ม OAuth ให้สัมพันธ์กับการ login เข้าสู่ระบบเดิมของ ODOO โดยมี 2 ผู้ให้บริการที่จำเป็นต้องเชื่อมต่อ นั่นก็คือ Google และ Facebook

Implement OAuth authentication that integrates with ODOO's existing login system, supporting two essential service providers:
- Google OAuth 2.0
- Facebook Login API

### 2. OdooAI Chat Integration | การเชื่อมต่อ OdooAI ในระบบแชท
ให้เพิ่มหรือปรับ group chat ของ ODOO ให้สามารถ invite OdooAI เข้ามาใน group เพื่อเริ่มสนทนาด้วยกันได้ 

Enhance ODOO's group chat functionality to allow inviting an AI assistant (OdooAI) to conversations.

#### OdooAI Behavior | พฤติกรรมของ OdooAI:
OdooAI จะเข้าร่วมสนทนาก็ต่อเมื่อ:
- ถูก invite เข้ามาในกลุ่ม
- เมื่อสมาชิกในกลุ่มเริ่มส่งข้อความโดยไม่ mention ถึงใครโดยเฉพาะ
- ถูก mention เช่น @OdooAI หรือ @everyone

OdooAI will participate in conversations when:
- It's invited to the group
- A group member sends a message without mentioning anyone specific
- It's mentioned directly (e.g., @OdooAI or @everyone)

OdooAI จะไม่เข้าร่วมบทสนทนาเมื่อ:
- มีการ mention ไปที่ผู้อื่นโดยเฉพาะเจาะจงโดยไม่มี OdooAI อยู่ในนั้น

OdooAI will not participate when:
- Messages mention specific individuals without including OdooAI

#### Context Access | การเข้าถึงบริบท:
OdooAI จะสามารถเข้าถึง topic หรือ history ของ group chat นั้นได้ เพื่อความเข้าใจในบริบทของบทสนทนาทั้งหมด

OdooAI should have access to the topic and history of the group chat to understand the full context of conversations.

## OPTIONAL FEATURES | คุณสมบัติเสริม
Future features to consider:

### Personal Customization | การปรับแต่งส่วนบุคคล
- Personalize OdooAI (ระบุความสนใจ หรือวิธีการในการตอบเช่น ตอบอย่างเป็นทางการ หรือสนุกสนาน)
- Allow users to customize AI responses (formal, casual, technical)

### System Control | การควบคุมระบบ
- Access and control ODOO system through AI assistant commands
- Implement permission levels for AI system actions

### External Integration | การเชื่อมต่อภายนอก
- Collaboration with third parties through standardized APIs
- Support for Model Context Protocol (MCP) for AI service integration

## PROJECT GOALS | เป้าหมายโครงการ
โปรเจคนี้เกิดขึ้นเพื่อให้นักศึกษาฝึกงานทั้งหมดได้:
- มีโอกาสทดลองทำงานร่วมกัน
- ได้รับการฝึกฝนการค้นคว้าข้อมูลที่เกี่ยวข้องกับงานที่ได้รับผิดชอบในอนาคต
- ได้ฝึกการบริหารจัดการภายในกลุ่ม
- ได้เก็บผลงานเป็นประวัติหรือ portfolio เป็นของตัวเอง

This project aims to provide intern students with opportunities to:
- Experience collaborative work in a development team
- Practice researching information related to their future responsibilities
- Learn group management and coordination
- Build a professional portfolio with tangible results
